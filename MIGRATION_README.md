# 資料庫遷移說明

## 概述
本次更新為試劑批號確認系統添加了兩個新欄位，並調整了欄位順序：
- **單位** (unit)
- **供應商名稱** (supplier)

**新的欄位排版順序：**
1. 檢驗項目名稱、批號、有效期限
2. 數量、單位、供應商名稱

## 新增欄位詳情

### 1. 單位 (unit)
- 類型：TEXT
- 用途：記錄試劑的計量單位
- 範例：瓶、盒、支、包等

### 2. 供應商名稱 (supplier)
- 類型：TEXT
- 用途：記錄試劑的供應商名稱
- 範例：台灣默克、西格瑪奧德里奇、羅氏診斷等

## 執行遷移步驟

### 步驟 1：執行遷移腳本
```bash
python migrate_db.py
```

### 步驟 2：檢查遷移結果
腳本會顯示：
- 現有欄位結構
- 新添加的欄位
- 最終表格結構

### 步驟 3：重啟應用程式
```bash
python app.py
```

## 影響的功能

### 1. 新增批號頁面
- 表單新增單位和供應商名稱欄位
- 兩個欄位都是可選填寫
- 欄位順序調整為：檢驗項目名稱、批號、有效期限 | 數量、單位、供應商名稱
- 移除備註欄位

### 2. 查詢批號頁面
- 查詢結果會顯示新欄位資訊
- 供應商和單位會以卡片形式展示

### 3. 批號列表頁面
- 表格新增供應商和單位欄位
- 詳細資訊模態框會顯示新欄位

### 4. 資料庫結構
- `used_reagents` 表格新增欄位
- 現有資料的單位和供應商欄位會設為 NULL
- 移除備註欄位

## 注意事項

1. **備份資料**：建議在執行遷移前備份現有資料庫
2. **向下相容**：新欄位為可選，不會影響現有功能
3. **資料完整性**：現有記錄的新欄位會自動設為預設值

## 回滾方案

如果需要回滾，可以執行以下 SQL 命令：
```sql
-- 刪除新欄位（注意：會遺失相關資料）
ALTER TABLE used_reagents DROP COLUMN unit;
ALTER TABLE used_reagents DROP COLUMN supplier;
-- 重新添加備註欄位
ALTER TABLE used_reagents ADD COLUMN note TEXT;
```

## 技術細節

- 使用 SQLite 的 `ALTER TABLE ADD COLUMN` 語法
- 新欄位會自動添加到表格末尾
- 現有資料的新欄位會設為 NULL 或預設值
